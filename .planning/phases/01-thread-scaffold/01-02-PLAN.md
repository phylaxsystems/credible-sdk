<objective>
Wire `StateWorkerThread` into the sidecar's `run_sidecar_once()` so the thread participates in
startup, shutdown, and failure detection alongside `EventSequencing`, `CoreEngine`, and
`TransactionObserver`.

Changes to `crates/sidecar/src/main.rs`:
1. Add `state_worker: Option<JoinHandle<...>>` field to `ThreadHandles`
2. In `join_all()`, join the state worker AFTER the engine (ordering matters — engine must send its
   final CommitHead before state worker can safely stop)
3. In `run_sidecar_once()`, construct and spawn `StateWorkerThread`
4. Add `sw_exited` oneshot receiver as a new arm in `run_async_components`'s `select!`
5. Add `handle_state_worker_exit()` handler following the same pattern as `handle_engine_exit()`

EthRpcSource is NOT modified — it must remain a registered fallback source.

Purpose: This closes the lifecycle loop. The sidecar can now detect state worker failure via the
oneshot channel and restart, and the state worker shuts down cleanly with SIGTERM.

Output: Modified `crates/sidecar/src/main.rs` — sidecar compiles and passes cargo check.
</objective>

fn handle_engine_exit(
    result: Result<
        Result<(), sidecar::engine::EngineError>,
        tokio::sync::oneshot::error::RecvError,
    >,
) {
    match result {
        Ok(Ok(())) => tracing::warn!("Engine exited unexpectedly"),
        Ok(Err(e)) => {
            let recoverable = ErrorRecoverability::from(&e).is_recoverable();
            record_error_recoverability(recoverable);
            if recoverable {
                tracing::error!(error = ?e, "Engine exited with recoverable error");
            } else {
                critical!(error = ?e, "Engine exited with unrecoverable error");
            }
        }
        Err(_) => tracing::error!("Engine notification channel dropped"),
    }
}
