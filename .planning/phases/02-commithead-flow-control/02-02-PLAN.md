<objective>
Add the CommitHead signal sender to CoreEngine: add `commit_head_tx: Option<flume::Sender<CommitHeadSignal>>` to the `CoreEngine` struct and `CoreEngineConfig`, extract a `send_commit_head_signal` helper that converts `U256` block number to `u64`, and call it on ALL return paths of `process_commit_head` including the early `!is_valid_state` path and the `NothingToCommit` error path.

Purpose: The state worker buffer (Plan 03) drains only on CommitHead signals. If any return path in `process_commit_head` skips the signal send, the buffer grows unboundedly during cache-invalidation events. The signal must be sent on every path — including early returns — so the state worker always knows the engine's current block number.

Output: Modified `engine/mod.rs` with the new field, config field, helper method, and signal sends on all `process_commit_head` return paths.

**Step 5: Add send_commit_head_signal helper method**

Add a private method to `CoreEngine<DB>` implementation block:
```rust
/// Send a CommitHead signal to the state worker thread.
///
/// Called on every return path from `process_commit_head` so the state worker
/// always knows the engine's current committed block number, even during
/// cache-invalidation events.
///
/// A `SendError` means the state worker receiver was dropped (state worker
/// exited or restarting). This is non-fatal — `EthRpcSource` covers the window.
fn send_commit_head_signal(&self, block_number: U256) {
    if let Some(ref tx) = self.commit_head_tx {
        let signal = CommitHeadSignal {
            block_number: block_number.saturating_to::<u64>(),
        };
        // Infallible for unbounded channel while receiver exists.
        // Ignore SendError — state worker may be restarting.
        let _ = tx.send(signal);
    }
}
```
