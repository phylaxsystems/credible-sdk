<objective>
Wire the CommitHead channel in `run_sidecar_once`: create `flume::unbounded::<CommitHeadSignal>()`, pass the sender to `CoreEngineConfig.commit_head_tx`, construct an `Arc<AtomicU64>` for the committed-head pointer, and pass the receiver plus config and atomic to `StateWorkerThread::new`. The state worker only spawns when all four config fields (ws_url, mdbx_path, genesis_path, state_depth) are present; if any are missing, the sidecar starts without it (EthRpcSource is still available as fallback).

Purpose: Plans 02 and 03 built all the components. This plan connects them. After this plan, the end-to-end flow is operational: engine processes a block, calls `process_commit_head`, sends `CommitHeadSignal` to the state worker, state worker flushes buffered updates to MDBX, and `committed_head` atomic is updated with Release ordering ready for Phase 3's `MdbxSource` to consume with Acquire ordering.

Output: `main.rs` wired with the channel, committed_head atomic, and conditional state worker spawn.
</objective>

// Phase 3 will wire committed_head to MdbxSource.
// Constructed here (not inside the thread) so both consumers share the same Arc.
// Arc<AtomicU64>: state worker writes with Release ordering after each flush.
// Initial value 0 — MdbxSource will fall back to EthRpcSource until first flush.

**Step 4: Conditionally spawn StateWorkerThread**

After the CoreEngine spawn and before `run_async_components`, replace the existing `StateWorkerThread::new(Arc::clone(&shutdown_flag)).spawn(...)` call with a conditional spawn based on config completeness:

```rust
// Spawn StateWorkerThread only when all required config fields are present.
// If any field is missing, the state worker is skipped; EthRpcSource remains the fallback.
let sw_exited = if config.state_worker.ws_url.is_some()
    && config.state_worker.mdbx_path.is_some()
    && config.state_worker.genesis_path.is_some()
    && config.state_worker.state_depth.is_some()
{
    let state_worker = StateWorkerThread::new(
        Arc::clone(&shutdown_flag),
        commit_head_rx,
        config.state_worker.clone(),
        Arc::clone(&committed_head),
    );
    let (sw_handle, sw_rx) = state_worker
        .spawn()
        .map_err(|e| anyhow::anyhow!("Failed to spawn state worker: {e}"))?;
    thread_handles.state_worker = Some(sw_handle);
    sw_rx
} else {
    tracing::warn!(
        "Embedded state worker not configured \
         (missing ws_url, mdbx_path, genesis_path, or state_depth); \
         relying on EthRpcSource for state"
    );
    // Create a dummy oneshot that never resolves — run_async_components will wait forever on it,
    // which is correct: the state worker never exits if it never starts.
    let (_, dummy_rx) = tokio::sync::oneshot::channel::<Result<(), sidecar::state_worker_thread::StateWorkerError>>();
    dummy_rx
};
```

Pass `sw_exited` to `run_async_components` as it already expects `sw_exited: oneshot::Receiver<...>`.
